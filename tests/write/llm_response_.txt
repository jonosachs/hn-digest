      "background": "In the early 2000s, much of the interactive content on the web—such as games and animations—ran on Adobe Flash. This was a 'proprietary' technology, meaning it was owned and controlled by one company (Adobe). To view Flash content, users had to install a 'browser plugin,' which is an additional piece of software that runs inside a web browser to handle specific file types that the browser cannot understand on its own. Flash used two main file types: .fla (the source project file containing all the artwork and code) and .swf (the finished, compressed file shown to users).\n\nFlash's primary advantage was its use of 'vector graphics.' Unlike 'raster graphics,' which are grids of colored pixels that get blurry when zoomed in, vector graphics are mathematical descriptions of shapes (like 'draw a circle with a 10-inch radius'). This makes the files very small and allows them to be resized perfectly to any screen. However, Flash was often criticized for being 'battery hungry' (using too much processing power, which drains mobile device batteries) and having security vulnerabilities that could allow hackers to attack a user's computer.\n\nThe industry eventually moved to 'Open Web Standards' like HTML5 and the Canvas API. These are sets of rules and technologies built directly into every modern browser that allow for high-performance graphics and interaction without needing a plugin. While this transition improved security and battery life, many of the specialized tools that artists used to create Flash content did not have modern equivalents, leading to the project described in the article."
